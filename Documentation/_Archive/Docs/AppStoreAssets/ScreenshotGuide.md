## Design Guidelines

### Visual Style
- **Use real app screenshots** (not mockups where possible)
- **Add device frames** for context (use Apple's device frames)
- **Consistent branding** across all screenshots
- **High contrast** text for readability
- **Professional polish** - no lorem ipsum or placeholder data

### Typography
- **Headline:** 48-64pt, Bold, App's primary font
- **Caption:** 28-36pt, Regular, Good contrast
- **Text color:** White on dark overlay or dark on light background
- **Readability:** Ensure 4.5:1 contrast ratio

### Device Frames
Use official Apple device frames:
- Download from Apple Design Resources
- Or use screenshot framing tools (Figma plugins, Sketch plugins)
- Maintain proper aspect ratios

### Captions
- **Position:** Bottom 1/3 of screenshot
- **Background:** Semi-transparent overlay (40% black or app color)
- **Length:** 2-3 lines maximum (150-200 characters)
- **Value-focused:** Tell users WHAT they get and WHY it matters

## Screenshot Production Workflow

### Step 1: Prepare App Data
1. Load app with realistic sample data
2. Configure perfect state for each screenshot
3. Ensure UI is in perfect state (no bugs, no placeholders)
4. Set device to optimal settings (time, battery, signal)

### Step 2: Capture Screenshots
1. Use Xcode Simulator for pixel-perfect screenshots
2. Cmd+S to save screenshots
3. Or use physical device with screenshot tool
4. Capture at highest resolution

### Step 3: Add Device Frames
1. Import screenshots into design tool (Figma, Sketch, Photoshop)
2. Add official Apple device frames
3. Ensure proper alignment and sizing

### Step 4: Add Headlines & Captions
1. Add headline text (bold, large)
2. Add caption below (smaller, 2-3 lines)
3. Add semi-transparent background for readability
4. Ensure text is accessible and readable

### Step 5: Export
1. Export at exact required dimensions
2. Use PNG format (high quality)
3. Verify file size < 5MB per screenshot
4. Name files descriptively: `screenshot_1_home.png`

### Step 6: Optimize
1. Compress PNGs (use TinyPNG or ImageOptim)
2. Maintain quality while reducing file size
3. Verify colors look correct
4. Test on different screens

## Localization Considerations

If supporting multiple languages:
- Create separate screenshots for each language
- Translate headlines and captions
- Ensure UI text is in correct language
- Account for text expansion (some languages need 30% more space)
- Test RTL languages (Arabic, Hebrew) separately

## App Store Optimization

### Screenshot Order Matters
- First 3 screenshots are most important (above the fold on many devices)
- Lead with most compelling features
- Tell a story: Problem → Solution → Benefits

### A/B Testing Strategy
Consider testing:
- Different headline styles
- Different feature priorities
- With/without captions
- Different color schemes
- With/without device frames
